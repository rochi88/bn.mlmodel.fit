# লাইকলিহুড (Likelihood) ও ম্যাক্সিমাম লাইকলিহুড এস্টিমেশন (MLE)

বার্নুলি ডিস্ট্রিবিউশন থেকে আমরা জানি আউটপুট যদি বাইনারি টাইপ হয় তাহলে আমরা সেটা এভাবে লিখতে পারি,

P(y=1∣x;θ)=hθ(x)P(y=0∣x;θ)=1−hθ(x) \begin{aligned} P(y = 1 | x; \theta) &= h_{\theta}(x) \\ P(y = 0 | x; \theta) &= 1 - h_{\theta}(x) \end{aligned} P(y=1∣x;θ)P(y=0∣x;θ)​=hθ​(x)=1−hθ​(x)​

একসাথে প্রকাশ করলে,

P(y∣x;θ)=(hθ(x))y(1−hθ(x))1−y P(y | x;\theta) = ( h_{\theta}(x) )^{y} (1 - h_{\theta}(x))^{1 - y} P(y∣x;θ)=(hθ​(x))y(1−hθ​(x))1−y

এটা হল, θ\thetaθ প্যারামিটারের মান ও ইনপুট xxx দেয়া থাকলে yyy এর ক্লাস কোনটা হওয়ার সম্ভাবনা সেটা প্রকাশ করে। কিন্তু সমস্যা হল, আমাদেরকে যে ডেটাসেট দেয়া হবে সেখান থেকে θ\thetaθ এর মান বের করতে হবে। আর সেটা বের করার জন্য আমাদের একটি গাণিতিক সমীকরণ দরকার যেটা কিনা মডেল করতে পারে, θ\thetaθ এর মানের সাথে xxx ও yyy এর সম্পর্ক কিরকম।

কয়েন টসের মত আমরা এই বিষয়টা চিন্তা করব, যেমন, কয়েন টসের আউটকাম এর সাথে এর আগের রেজাল্টের কোন সম্পর্ক নেই। তারমানে আমাদের যদি mmm সংখ্যক ডেটাসেট দেয়া হয় তাহলে আমরা ধরে নেব প্রতিটি ডেটাসেট ইন্ডিপেন্ডেন্টলি (Independence rule of Probability) জেনারেটেড।

আবারও কয়েন টসে ফিরে গেলে, আমি যদি একটা কয়েনকে 10 বার টস করি, তাহলে এই 10 টসের রেজাল্ট আমি লিখে রাখি, তাহলে পরবর্তী টসের রেজাল্ট কি হবে সেটা কি অনুমান করতে পারব? হ্যাঁ পারা যায়, যদি 7 টা রেজাল্ট আসে Head এবং 3 টা আসে Tail তাহলে আমি বলব পরবর্তী টসে হেড হওয়ার সম্ভাবনা 70%। এটা বের করলাম এভাবে,

θ=Number of HeadsTotal Toss Count=710 \theta = \frac{\text{Number of Heads}}{\text{Total Toss Count}} = \frac{7}{10} θ=Total Toss CountNumber of Heads​=107​

এটাই আমার নির্ণেয় প্যারামিটার।

আমি ধরে নিচ্ছি hθ(x)=θh_{\theta}(x) = \thetahθ​(x)=θ , অর্থাৎ আমার হাইপোথিসিস আউটপুট এবং থিটার মান একই হবে। যেমন GLM এর ক্ষেত্রে আমি সিগময়েড নিয়েছিলাম।

অর্থাৎ, Head হওয়ার সম্ভাবনা, θ=0.7\theta = 0.7θ=0.7 হলে, Tail হওয়ার সম্ভাবনা, θ=0.3\theta = 0.3θ=0.3।

গাণিতিকভাবে যদি প্রকাশ করি,

X=[H,H,H,H,H,H,H,T,T,T]P(y=H∣X;θ=0.7)=hθ(X)=0.7P(y=T∣X;θ=0.7)=1−hθ(X)=1−0.7=0.3 \begin{aligned} X &= [H, H, H, H, H, H,H, T, T, T] \\ P(y = H | X ; \theta=0.7) &= h_{\theta}(X) = 0.7 \\ P(y = T | X; \theta = 0.7) &= 1 - h_{\theta}(X) = 1 - 0.7 = 0.3 \end{aligned} XP(y=H∣X;θ=0.7)P(y=T∣X;θ=0.7)​=[H,H,H,H,H,H,H,T,T,T]=hθ​(X)=0.7=1−hθ​(X)=1−0.7=0.3​

θ\thetaθ এর মান বের করা সহজ কারণ আমাদের সমস্যার ডাইমেনশন মাত্র একটা। এই মান আরেকভাবে বের করা যায়, সেটা হল ম্যাক্সিমাম লাইকলিহুড এস্টিমেশনের (MLE) মাধ্যমে।

বিষয়টা হল θ\thetaθ এর মান এমন হবে যেন ইনপুট XXX ও yyy আউটপুটের জন্য লাইকলিহুড ম্যাক্সিমাম হয়। অর্থাৎ, আমরা যেহেতু জানিনা কয়েনটা আদৌ বায়াজড না ফেয়ার কয়েন, তো আমরা ধরে নিব HHH আসার সম্ভাবনা θ\thetaθ এবং TTT আসার সম্ভাবনা 1−θ1 - \theta1−θ ।

এইযে দেয়া ডেটাসেট এ 777 টা HeadHeadHead এবং 333 টা TailTailTail হবে, সেটা কিন্তু N!S!×(N−S)!=10!7!×3!=120\frac{N!}{S! \times (N-S)!} = \frac{10!}{7! \times 3!} = 120S!×(N−S)!N!​=7!×3!10!​=120 উপায়ে হতে পারে। (Binomial Distribution, N = Number of trials, S = Number of Success\text{N = Number of trials, S = Number of Success}N = Number of trials, S = Number of Success)

এখানে আসলে ফ্যাক্টোরিয়ালসহ যে টার্ম আছে সেটার দরকার নেই, কারণ এর ডেরিভেটিভ সমান শূন্য ধরে সল্ভ করলে কন্সট্যান্ট টার্ম বাতিল হয়ে যাবে।

প্রব্যাবিলিটি অনেক ছোট একটা ফ্র্যাকশন, এটাকে বার বার গুণ করতে থাকলে নিউমেরিকাল আন্ডারফ্লো হয় (Numerical Underflow)। সেটা রোধ করার জন্য, আমরা সবসময় লাইকলিহুডের লগ নিয়ে থাকি ।

এখন সমস্যা একটাই, বের করতে হবে θ\thetaθ এর কোন মানের জন্য ℓ(θ)\ell {(\theta)}ℓ(θ) এর মান সবচেয়ে বেশি হয়। এবং এটাই আমাদের সেই লস ফাংশন।

যখনই ম্যাক্সিমাইজেশন বা মিনিমাইজেশনের সমস্যা চলে আসে, তখনই আমরা সেই ফাংশনের ডেরিভেটিভ নেই, কারণ ফাংশনের সর্বোচ্চ/সর্বনিম্ন বিন্দুতে ওই ফাংশনের ডেরিভেটিভের মান শূন্য।

অল্প কয়েকটা প্যারামিটার থাকলে এই মান বের করা সহজ কিন্তু প্যারামিটার যদি হাজার হাজার হয় তাহলে নিউমেরিক্যাল ইটারেটিভ সল্যুশনে এইসব সমস্যা সল্ভ করা যায়।

# Creating weight vector [without bias or Theta_0 or let's call it W_0]

# Sigmoid function [non-optimized, overflow might occur]
def _sigmoid(z):
    return 1.0 / (1 + math.exp(-z))

# প্রেডিকশন

hθ(x)=σ(θTx)=σ(∑jθjxj) \begin{aligned} h_{\theta}(x) &= \sigma(\theta^{T}x) \\ &= \sigma \left( \sum_{j} \theta_{j}x_{j} \right) \end{aligned} hθ​(x)​=σ(θTx)=σ(j∑​θj​xj​)​

প্রথমটা ভেক্টরাইজড ও দ্বিতীয়টি নন-ভেক্টরাইজড।

# Function for prediction
def _predict(x, w):
    _sum = 0
    for _x, _w in zip(x, w):
        _sum += _x * _w
    return _sigmoid(_sum)

# লস কম্পিউটেশন

J(θ)=−1m∑i=1my(i)log⁡h(x(i))+(1−y(i))log⁡(1−h(x(i))) J(\theta) = - \frac{1}{m} \sum_{i=1}^{m} y^{(i)} \log {h(x^{(i)})} + (1 - y^{(i)}) \log { \left(1 - h(x^{(i)}) \right)} J(θ)=−m1​i=1∑m​y(i)logh(x(i))+(1−y(i))log(1−h(x(i)))
# Function for computing loss
def _computeCost(X, W, y):
    # Number of data / rows
    m = len(X)

    #loss [obvious!]
    loss = []

    # Computing the prediction
    yHat = [_predict(X[i], W) for i in range(m)]

    # Clipping the prediction between 0.01 and 0.99 to avoid 'divide by zero'
    for i, _y in enumerate(yHat):
        if (_y == 1):
            yHat[i] = 0.99
        elif (_y == 0):
            yHat[i] = 0.01
        else:
            pass

    # Computing the loss using log-loss
    for _y, _yHat in zip(y, yHat):
        log_loss = _y * math.log(_yHat) + (1 - _y) * math.log(1 - _yHat)
        loss.append(log_loss)

    return -sum(loss) / (len(loss))

# গ্রেডিয়েন্ট কম্পিউটেশন

∂J(θ)∂θ=(y−hθ(x))×x \frac{\partial J(\theta)}{\partial \theta} = (y - h_{\theta}(x)) \times x ∂θ∂J(θ)​=(y−hθ​(x))×x
# Takes one row of x and y
def _computeGradient(x, W, y):

    if (type(y) == np.ndarray):
        raise TypeError("'y' can't be array")

    yHat = _predict(x, W)

    # Gradient array
    gradW = []

    # Computing the gradients
    for _x in x:
        # For gradient ascent y - yHat
        # For gradient descent yHat - y
        gradW.append( (y - yHat) * _x )

    return np.array(gradW)

# গ্রেডিয়েন্ট অ্যাসেন্ট ইম্প্লিমেন্টেশন

θj:=θj+α×(y−hθ(x))×x \theta_{j} := \theta_{j} + \alpha \times (y - h_{\theta}(x)) \times x θj​:=θj​+α×(y−hθ​(x))×x
# Implementation of gradient ascent
costs = []
iteration = 100
lr = .0001
for k in range(iteration):
    for x, _y in zip(X, y):
        gradw = _computeGradient(x, W, _y)
        for i, w in enumerate(W):
            W[i] = W[i] + lr * gradw[i]
    cost = _computeCost(X, W, y)
    costs.append(cost)
    print("COST:  {}".format(cost))

# NumPy ব্যবহার করে ভেক্টরাইজড লজিস্টিক রিগ্রেশন মডেল

সব ইক্যুয়েশন আগের মতই, এখন শুধু ম্যাট্রিক্স অপারেশন করে একই জিনিস করা হবে।

def sigmoid(z):
    return 1.00 / ( 1 + np.exp(-z))

def predict(X, W):
    return sigmoid(np.dot(X, W))

def computeGradient(X, y, W):
    yHat = predict(X, W)
    return np.dot(y - yHat, X)

def computeCost(X, y, W):
    yHat = np.clip(predict(X, W), 0.01, 0.99)
    first_term = y * np.log(yHat)
    second_term = (1 - y) * np.log(1 - yHat)
    log_loss = - np.mean(first_term + second_term)
    return log_loss

# Implementation of gradient ascent
W = np.ones(image_size)
costs = []
iteration = 100
lr = .0001
print("COST : {}".format(computeCost(X, y, W)))
for i in range(iteration):
    gradW = computeGradient(X, y, W)
    W = W  + lr * gradW
    print("COST : {}".format(computeCost(X, y, W)))
